## 1. Начало работы

### 1.1 Регистрация

1. Откройте сайт в браузере.
2. Нажмите **«Зарегистрироваться»** на странице входа.
3. Заполните поля:
   - **Имя** — как к вам обращаться
   - **Email** — на него придёт письмо с подтверждением
   - **Пароль** — минимум 8 символов
4. Нажмите **«Зарегистрироваться»**.
5. Вы **автоматически входите** в систему — дополнительный вход не требуется.
6. На указанный email приходит письмо с просьбой подтвердить адрес.
7. Вверху страницы появится жёлтая плашка **«Пожалуйста, подтвердите email»** с указанием срока (24 часа) и кнопкой повторной отправки письма.
8. Если не подтвердить email в течение 24 часов, учётная запись будет **заблокирована**. Для восстановления доступа можно повторно запросить письмо или сменить email в настройках.

> Если письмо не пришло — проверьте папку «Спам». Повторную отправку можно запросить из плашки подтверждения или в разделе Настройки → Безопасность.

### 1.2 Вход в систему

1. Введите email и пароль.
2. Нажмите **«Войти»**.
3. При неверных данных отображается ошибка **«Неверный email или пароль»**.
4. Если email не был подтверждён в течение 24 часов — отображается ошибка **«Учётная запись заблокирована»** (перенаправление на страницу блокировки, где можно повторно запросить письмо).
5. Если превышен лимит устройств для вашего тарифа — отображается соответствующая ошибка.

### 1.3 Восстановление пароля

1. На странице входа нажмите **«Забыли пароль?»**.
2. Введите email, к которому привязан аккаунт.
3. Откройте письмо. В нём два способа сброса:
   - **Способ 1:** перейдите по прямой ссылке из письма.
   - **Способ 2:** введите 6-значный код из письма на странице сброса пароля.
4. Придумайте новый пароль и подтвердите его.
5. После сброса все активные сессии будут завершены — потребуется войти заново.

### 1.4 Подтверждение email

- Сразу после регистрации вверху страницы показывается плашка с предупреждением и дедлайном.
- Если письмо не пришло — нажмите **«Отправить повторно»** (доступно из плашки и в Настройки → Безопасность).
- **Важно:** без подтверждения email учётная запись будет **заблокирована через 24 часа**.
- Если аккаунт заблокирован — откроется страница блокировки, где можно повторно запросить письмо или выйти.

## 2. Главная

После входа вы попадаете на страницу **«Главная»** (`/files`). Это стартовая страница сервиса.

### 2.1 Зона загрузки

- Нажмите на область загрузки или перетащите файлы мышью.
- Дождитесь окончания загрузки — появится зелёная отметка.
- Файл появится в блоке «Добавлен новый файл» и в списке последних файлов.

**Отмена загрузки:**
- Во время загрузки нажмите кнопку **«Отменить»** — файл будет удалён и не появится в вашем списке.

### 2.2 Добавление ссылки

Вместо загрузки файла можно сохранить веб-ссылку:

1. Вставьте URL в форму **«Добавить ссылку»**.
2. Нажмите **«Предпросмотр»** — система покажет заголовок, описание и картинку с сайта.
3. Нажмите **«Сохранить»** — ссылка появится как файл в вашем списке.

### 2.3 Добавлен новый файл

- Сразу после загрузки показывается карточка с именем, размером и датой.
- Можно сразу добавить описание к файлу.
- Кнопка **«Открыть файл»** ведёт на полную карточку файла.

### 2.4 Последние файлы

В правой части страницы отображается блок **«Последние файлы»** — недавно добавленные или полученные файлы. Нажмите на файл, чтобы открыть его карточку. Ссылка **«Все файлы»** ведёт на страницу «Папки и Файлы».

## 3. Папки и Файлы

Страница **«Папки и Файлы»** (`/folders`) — полноценный проводник для работы с файлами и папками.

### 3.1 Интерфейс

- **Верхняя панель** — переключение между **Локальными** (ваши папки) и **Общими** папками, вкладки фильтра, поиск, действия.
- **Центральная область** — список файлов и папок (таблица или сетка).
- **Хлебные крошки** — навигация по вложенным папкам.
- **Счётчик хранилища** — отображается справа вверху.

### 3.2 Фильтры и поиск

**Вкладки фильтра (сверху):**

| Вкладка | Что показывает |
|---------|---------------|
| **Все** | Все файлы, к которым у вас есть доступ |
| **Мои файлы** | Файлы, которые вы загрузили |
| **Полученные** | Файлы, которыми с вами поделились |
| **Избранное** | Файлы, отмеченные звёздочкой |

**Поиск и сортировка:**

- Строка поиска — ищет по названию файла.
- Фильтр по тегу — выберите тег из выпадающего списка.
- Сортировка: по дате, по типу (расширению), по размеру.
- Порядок: по возрастанию / по убыванию.

**Фильтр по типу файла:**

В виде кнопок под строкой поиска: Изображения, Видео, Аудио, Документы, Архивы, Ссылки, Другое.

### 3.3 Вид

- **Таблица** — колонки: имя, дата изменения, размер, действия.
- **Сетка** — карточки с иконкой/превью и именем файла.

### 3.4 Работа с папками

- **Создать** — кнопка «Новая папка» на верхней панели.
- **Переименовать** — в контекстном меню (три точки) у папки.
- **Удалить** — пустую папку можно удалить сразу; если в ней есть файлы — потребуется подтверждение принудительного удаления.
- **Заметки к папке** — для любой локальной папки доступен блок личных заметок (подробнее в разделе [«Комментарии»](#5-комментарии)).
- Навигация — переход по папкам через хлебные крошки или двойной клик.

### 3.5 Работа с файлами

У файлов, для которых включено **версионирование**, в списке отображается значок **V**.

**Контекстное меню** (три точки у файла):

- Открыть (переход на карточку файла)
- Скачать
- В избранное
- Добавить тег
- Переместить в папку
- Удалить

**Массовые операции:**

1. Отметьте несколько файлов чекбоксами.
2. Внизу появится панель с выбором действия:
   - **Назначить тег** — выберите или создайте тег для всех выбранных файлов
   - **Переместить в папку** — выберите целевую папку
   - **Удалить** — подтвердите массовое удаление
3. Снимите выделение кнопкой **«Снять выделение»**.

### 3.6 Загрузка в папку

Находясь внутри папки, нажмите **«Добавить»** на верхней панели. Откроется модальное окно с двумя вкладками:

- **Файл** — выберите или перетащите файл для загрузки.
- **Ссылка** — вставьте URL, сделайте предпросмотр и сохраните.

## 4. Карточка файла

Нажмите на любой файл, чтобы открыть его детальную карточку (`/files/:id`).

### 4.1 Просмотр и метаданные

- Для изображений и видео — показывается превью.
- Для URL-файлов — показывается заголовок, описание и картинка с сайта.
- Отображаются: имя, размер, MIME-тип, статус, дата загрузки.
- Если вы владелец — отметка **«Владелец»**.

### 4.2 Действия

- **Скачать** — загрузить файл на устройство.
- **Смотреть в браузере** — для изображений, видео, аудио, PDF.
- **В избранное** / **Убрать из избранного** — отметка звёздочкой.
- **Поделиться** — открыть диалог отправки контакту.
- **Создать ссылку** — создать публичную ссылку на файл.
- **Добавить в общую папку** — прикрепить файл к одной из общих папок (если вы владелец или получатель файла).
- **Добавить в мои файлы** — для файлов, доступных только в общей папке.
- **Убрать из моих файлов** / **Удалить** — для полученных файлов или владельца соответственно.

### 4.3 Описание

Текстовое поле (до 500 символов). Нажмите **«Сохранить»**, чтобы применить.

### 4.4 Теги

- Просмотр текущих тегов.
- Кнопка **+** для добавления нового тега — поиск по существующим или создание нового.
- Крестик на теге — удаление тега с файла.

### 4.5 Перемещение в папку

- Выберите папку из выпадающего списка.
- Нажмите **«Сохранить»** — файл переместится в указанную папку.

### 4.6 Активные ссылки

На правой боковой панели отображаются все активные публичные ссылки на файл. Для каждой ссылки:

- URL (можно скопировать)
- Срок действия
- Статус (активна/отключена/просрочена)
- Флаг «Разрешено сохранение»
- Кнопка **«Отключить»** — деактивировать ссылку досрочно.

### 4.7 Кто имеет доступ

Список пользователей и тип доступа (владелец, получен, сохранён).

### 4.8 Активность

Хронология действий с файлом: загрузка, скачивание, отправка, создание ссылок, добавление в избранное, перемещение, удаление и т.д.

### 4.9 Версионирование файлов

Для файлов, которыми вы владеете, доступно управление версиями. Это позволяет хранить историю изменений файла и при необходимости откатываться к предыдущей версии.

**Включение версионирования:**

При загрузке новой версии файла версионирование включается автоматически. Текущий файл сохраняется как версия **v1**, а новая загрузка становится следующей версией.

**Интерфейс версионирования:**

На карточке файла, если версионирование активно, отображается:
- **Селектор версий** — строка с переключателями между версиями. При выборе версии обновляется имя, размер и превью файла.
- **Значок V** — рядом с именем файла в карточке и в списке файлов.

**Добавление новой версии:**

1. Откройте карточку файла.
2. Нажмите **«Добавить версию»**.
3. Выберите или перетащите новый файл (того же типа, что и оригинал — изображение, видео, аудио или документ).
4. Дождитесь окончания загрузки — версия появится в списке.
5. Система обновит метаданные файла (название, размер) на основе новой версии.

> При первой загрузке версии исходный файл автоматически сохраняется как v1.

**Управление версиями:**

В блоке **«Версии файла»** на карточке (доступно только владельцу) можно:

- **Общее имя** — задать отображаемое имя для всех версий (поле «Общее имя»).
- **Редактировать версию** — изменить метку (например, «2.1», «Черновик») и добавить комментарий «Что изменилось?».
- **Деактивировать / активировать** — скрыть версию из общего списка.
- **Скачать** — загрузить конкретную версию на устройство.

**Скачивание определённой версии:**

1. В селекторе версий выберите нужную версию.
2. Нажмите кнопку **«Скачать»** — будет загружена именно выбранная версия.

**Ограничения:**

- Версионирование доступно только для файлов (не для URL-ссылок).
- Тип файла (MIME-группа) должен совпадать с оригиналом (изображение → изображение, видео → видео и т.д.).
- Добавлять версии может только владелец файла.

### 4.10 Комментарии к файлу

В нижней части карточки файла находится секция **«Комментарии»** с двумя вкладками:

- **Общие** — комментарии, видимые всем пользователям, имеющим доступ к файлу.
- **Мои заметки** — личные заметки, видимые только вам.

Для владельца файла доступен блок настроек: включение/отключение общих комментариев и политика переопределения для общей папки.

Подробнее — в разделе [«Комментарии»](#5-комментарии).

## 5. Комментарии

В DeliFile реализована система комментариев для файлов и папок. Комментарии бывают двух типов:

- **Общие (shared)** — видны всем пользователям, у которых есть доступ к объекту.
- **Личные заметки (private)** — видны только автору.

### 5.1 Где доступны комментарии

| Объект | Общие комментарии | Личные заметки |
|--------|:-:|:-:|
| Файл (вне общей папки) | Да | Да |
| Файл (в общей папке) | Да, если политика разрешает | Да |
| Общая папка | Да, если не запрещено настройками | Да |
| Локальная папка | Нет | Да |

### 5.2 Интерфейс

Комментарии отображаются в виде ленты с двумя вкладками: **«Общие»** и **«Мои заметки»**.

**Основные возможности:**

- **Написать комментарий** — введите текст в поле ввода и отправьте.
- **Ответить** — нажмите «Ответить» на любом корневом комментарии (один уровень вложенности).
- **Редактировать** — изменить свой комментарий после отправки.
- **Удалить** — удалить свой комментарий.
- **Непрочитанные** — у комментариев отображается счётчик непрочитанных (число новых комментариев с момента последнего просмотра).

### 5.3 Управление настройками

**Для файла вне общей папки:**
Владелец файла может включить или отключить общие комментарии через переключатель **«Разрешить общие комментарии»**.

**Для файла в общей папке:**
Владелец файла может задать политику переопределения:

| Режим | Результат |
|-------|-----------|
| `inherit` | Действует правило общей папки |
| `enabled` | Общие комментарии разрешены, даже если общая папка их запрещает |
| `disabled` | Общие комментарии запрещены, даже если общая папка их разрешает |

**Для общей папки:**
Владелец общей папки может выбрать режим в диалоге управления доступом:

| Режим | Комментарии к папке | Комментарии к файлам внутри |
|-------|:-:|:-:|
| `enabled` | Разрешены | По умолчанию разрешены |
| `disabled` | Запрещены | По умолчанию запрещены |
| `inherit_for_items` | Разрешены | Файлы наследуют правило папки |

### 5.4 Контексты отображения

- **В карточке файла** (`/files/:id`) — секция «Комментарии» внизу.
- **В локальной папке** (`/folders`) — кнопка «Заметки к папке».
- **В общей папке** (`/folders`, вкладка «Общие») — кнопка «Обсуждение».

### 5.5 Упоминания

В тексте комментария можно упомянуть другого пользователя через `@`. Упомянутый пользователь получит push-уведомление. Упоминать можно только пользователей, у которых есть доступ к обсуждаемому объекту.

### 5.6 Уведомления

При создании нового комментария или ответе участники обсуждения получают push-уведомления (если уведомления включены в настройках).

## 6. Теги

Теги — способ организации файлов с помощью произвольных меток. Управление тегами вынесено в отдельный пункт меню **«Теги»** (`/tags`).

### 6.1 Управление тегами

- **Создать тег** — введите название (до 50 символов) и нажмите «Добавить».
- **Переименовать** — нажмите на имя тега, введите новое и сохраните.
- **Удалить** — подтвердите удаление (тег отвяжется от всех файлов).
- **Поиск** — фильтрация списка тегов по названию.
- **Просмотр файлов** — нажмите на счётчик файлов у тега, чтобы перейти к списку файлов с этим тегом.

### 6.2 Назначение тегов файлам

Теги можно назначать тремя способами:

1. **Из карточки файла** — откройте файл, нажмите «+» в блоке тегов, найдите или создайте тег.
2. **Из контекстного меню** — в проводнике нажмите «Добавить тег» у файла.
3. **Массово** — выберите несколько файлов и в панели массовых операций выберите «Назначить тег».

### 6.3 Фильтрация по тегу

В проводнике (`/folders`) доступен выпадающий список **«Все теги»** — выберите нужный тег, чтобы увидеть только файлы с этой меткой.

## 7. Шеринг файлов

### 7.1 Поделиться с контактом

1. Откройте карточку файла.
2. Нажмите **«Поделиться»**.
3. Выберите контакт из адресной книги.
4. Файл появится в разделе «Полученные» у выбранного контакта.
5. Контакт получит push-уведомление (если включено).

> Если контакт ещё не зарегистрирован в системе — файл будет доставлен автоматически после его регистрации и принятия приглашения.

### 7.2 Отозвать доступ

1. В карточке файла нажмите **«Доступ»** (правая панель).
2. Найдите контакт в списке — доступ будет отозван.

### 7.3 Публичная ссылка

1. Откройте карточку файла.
2. Нажмите **«Создать ссылку»**.
3. Настройте параметры:
   - **Срок действия** — через сколько часов ссылка перестанет работать (от 1 до 720 часов)
   - **Разрешить сохранение** — если включено, другие пользователи смогут сохранить файл к себе
4. Нажмите **«Создать»**.
5. Скопируйте получившуюся ссылку и отправьте кому угодно.

### 7.4 Просмотр по ссылке (неавторизованные пользователи)

По ссылке любой человек может:
- увидеть название и размер файла
- скачать файл
- сохранить файл в свой аккаунт (если разрешено и есть учётная запись)

### 7.5 Управление ссылками

В карточке файла на правой панели в разделе **«Активные ссылки»** вы можете:
- посмотреть все созданные ссылки (для каждой указан создатель)
- скопировать любую ссылку
- отключить ссылку досрочно (только создатель ссылки)

## 8. Общие папки

Общие папки позволяют работать с файлами нескольким пользователям совместно.

### 8.1 Создание общей папки

1. Перейдите на страницу **«Папки и Файлы»** (`/folders`).
2. В левой боковой панели выберите вкладку **«Общие»**.
3. Нажмите **«Новая папка»** и задайте название.

### 8.2 Управление доступом

1. Откройте общую папку и нажмите **«Управление доступом»** в контекстном меню.
2. Нажмите **«Добавить»**.
3. Выберите контакт и уровень доступа:
   - **Просмотр** — может только смотреть и скачивать файлы
   - **Редактирование** — может загружать, удалять и изменять файлы
4. Можно удалить участника или изменить его уровень доступа.
5. В диалоге управления доступом также доступен блок **«Комментарии»** с настройкой `shared_comments_mode` (подробнее в разделе [«Комментарии»](#5-комментарии)).

### 8.3 Обсуждение общей папки

У общей папки есть панель **«Обсуждение»** (кнопка в интерфейсе папки). В ней доступны:

- **Общие комментарии** — видимые всем участникам папки. Если `shared_comments_mode` установлен в `disabled`, общие комментарии к папке запрещены.
- **Мои заметки** — личные заметки, видимые только вам.
- **Настройка политики:** владелец папки может выбрать режим `Разрешены`, `Запрещены` или `Наследуются файлами`.

Подробнее — в разделе [«Комментарии»](#5-комментарии).

### 8.4 Загрузка в общую папку

1. Откройте общую папку (двойной клик).
2. Нажмите **«Добавить»** → выберите **«Файл»** или **«Ссылка»**.
3. Загруженные файлы будут доступны всем участникам.

### 8.5 Добавление в «Мои файлы»

Если вы участник общей папки и хотите, чтобы файл был всегда под рукой:
1. Нажмите на файл.
2. Выберите **«Добавить в мои файлы»**.
3. Файл появится в вашем личном списке.

### 8.6 Публичная ссылка на общую папку

Как и для обычных файлов, можно создать публичную ссылку на всю общую папку с настройкой срока действия и уровня доступа.

Если вы авторизованы и перешли по публичной ссылке на общую папку — доступ к папке будет предоставлен **автоматически** (с уровнем, указанным в ссылке). Повторный запрос доступа не требуется.

### 8.7 Вложенные папки

В общих папках поддерживается вложенность:
1. Откройте общую папку, чтобы увидеть её содержимое и подпапки.
2. Вверху отображается **хлебная крошка** (breadcrumb) — нажмите на любой уровень для быстрого перехода.
3. Нажмите **«Добавить подпапку»**, чтобы создать вложенную папку (доступно участникам с правом редактирования).
4. Права доступа наследуются от корневой папки — если у вас есть доступ к корню, он распространяется на все вложенные папки.

### 8.8 Выход из общей папки

1. Откройте общую папку.
2. В контекстном меню нажмите **«Покинуть папку»**.
3. Вы будете удалены из участников папки и потеряете доступ ко всем её файлам.

> Владелец общей папки не может покинуть папку — он может только удалить её полностью.

### 8.9 Удаление файла из общей папки

Удалить файл из общей папки могут:
- **Владелец файла** — может удалить свой файл из папки в любой момент.
- **Участник с правом «Редактирование»** — может удалить любой файл из папки (в том числе загруженный другим участником).

Файл при этом отвязывается от общей папки, но остаётся в личных файлах того, кто его загрузил (если он не был помечен как «Только в общей папке»).

## 11. Редактор Markdown

В DeliFile можно создавать и редактировать Markdown-документы прямо в браузере. Доступен WYSIWYG-редактор на основе Tiptap с подсветкой синтаксиса, автосохранением и поддержкой изображений.

### 11.1 Создание Markdown-документа

1. На странице **«Папки и Файлы»** откройте карточку файла.
2. Если файл — это Markdown-документ, внизу отобразится секция **«Markdown-редактор»**.
3. Если документа ещё нет, нажмите **«Создать»**, введите имя файла и нажмите **«Создать документ»**.
4. Расширение `.md` добавится автоматически, если не указано.
5. Документ появится в списке файлов с иконкой Markdown.

### 11.2 Редактирование

1. Нажмите на Markdown-документ в списке файлов.
2. В карточке файла разверните секцию **«Markdown-редактор»** или нажмите **«Открыть редактор»** для полноэкранного режима.
3. Используйте **панель инструментов** для форматирования:

   | Кнопка | Действие |
   |--------|----------|
   | **B** | Жирный текст (`**bold**`) |
   | *I* | Курсив (`*italic*`) |
   | ~~S~~ | Зачёркнутый |
   | `</>` | Встроенный код |
   | **H1**, **H2**, **H3** | Заголовки разных уровней |
   | • | Маркированный список |
   | 1. | Нумерованный список |
   | ☐ | Список задач (Task list) |
   | ❝ | Цитата (Blockquote) |
   | `{}` | Блок кода |
   | ↩ | Отмена (Undo) |
   | ↪ | Повтор (Redo) |
   | 🖼 | Вставить изображение |

4. Изменения автоматически сохраняются через **3 секунды** после остановки ввода (автосохранение).
5. Можно также нажать **«Сохранить»** вручную (или `Ctrl+S`).

### 11.3 Вставка изображений

1. Нажмите кнопку **🖼** на панели инструментов.
2. Откроется окно выбора изображения — в нём отображаются все ваши изображения и расшаренные вам.
3. Используйте строку поиска для фильтрации (с задержкой 400мс).
4. Выберите изображение и нажмите **«Вставить»**.
5. Изображение появится в редакторе. При выделении изображения появляется **панель изменения размера**: S (200px), M (400px), L (640px) или произвольное значение в пикселях.
6. Изображения сохраняются со стабильными ссылками — они будут корректно отображаться после перезагрузки страницы.

### 11.4 Одновременное редактирование (блокировки)

Чтобы два человека случайно не редактировали документ одновременно, используется механизм **блокировок**:

- При открытии редактора система автоматически **захватывает блокировку** (pessimistic lock).
- Блокировка **продлевается каждые 60 секунд** (heartbeat).
- Если блокировка истекла — появится баннер **«Сессия редактирования истекла»**.
  Нажмите **«Продолжить редактирование»**, чтобы восстановить блокировку.
- Если блокировку перехватил владелец документа — появится баннер **«Документ забран пользователем [имя]»**.
  Редактирование станет доступным только для чтения.
- Если документ редактирует другой пользователь — при открытии вы увидите баннер **«Документ редактирует [имя]»**.
  Владелец может нажать **«Забрать документ»**, чтобы принудительно перехватить блокировку.
- При закрытии редактора блокировка автоматически снимается.

Состояния редактора:

| Статус | Редактирование | Описание |
|--------|:-:|----------|
| Есть блокировка | ✅ | Можно редактировать |
| Нет блокировки (read-only) | ❌ | Только просмотр |
| Блокировка истекла | ❌ | Нажмите «Продолжить» |
| Перехвачена владельцем | ❌ | Обратитесь к владельцу |

### 11.5 Автосохранение

- Редактор автоматически сохраняет изменения через **3 секунды** после последнего ввода.
- Во время сохранения отображается индикатор **«Сохранение...»**.
- При конфликте версий (документ был изменён в другой сессии) появляется баннер **«Конфликт версий»** — требуется ручное разрешение.
- При превышении квоты хранилища — ошибка **«Превышен лимит хранилища»**.

### 11.6 Просмотр документа

- Если у вас нет прав на редактирование, документ открывается в **режиме только для чтения**.
- Весь контент (текст, изображения, списки) отображается, но панель инструментов скрыта.
- Вы по-прежнему можете скопировать текст и скачать оригинал файла.

## 16. Часто задаваемые вопросы

**Q: Какой максимальный размер файла можно загрузить?**
Зависит от тарифа: Free — 50 МБ, Silver — 100 МБ, Gold — 150 МБ.

**Q: Как долго хранятся файлы?**
Файлы хранятся, пока вы их не удалите, или пока не истечёт срок их действия (если он был установлен).

**Q: Может ли кто-то увидеть мои файлы без моего разрешения?**
Нет. S3-хранилище полностью приватное. Доступ к файлу есть только у вас и тех, кому вы явно дали доступ.

**Q: Как удалить файл окончательно?**
После удаления файл помечается как удалённый. Система полностью очищает его в течение часа.

**Q: Что происходит, когда заканчивается место?**
Вы не сможете загружать новые файлы, пока не освободите место или не перейдёте на тариф с большим объёмом.

**Q: Как отозвать доступ к файлу?**
Откройте карточку файла → правая панель «Кто имеет доступ» → нажмите «Отозвать» рядом с контактом.

**Q: Можно ли восстановить удалённый файл?**
Нет, удаление необратимо. Будьте внимательны.

**Q: Как выйти из аккаунта на всех устройствах?**
Настройки → Безопасность → «Завершить все сессии».

**Q: Безопасно ли сохранять ссылки на файлы?**
Да. Ссылки защищены уникальным токеном. Вы можете отключить любую ссылку в любой момент.

## 17. Полученные (Inbox)

Страница **«Полученные»** (`/inbox`) доступна, когда в настройках отключено автоматическое добавление полученных файлов. Здесь вы управляете файлами и общими папками, которыми с вами поделились.

### 17.1 Вкладка «Файлы»

- Отображает список файлов, которыми с вами поделились.
- Для каждого файла указано: название, отправитель, дата.
- **Чекбоксы** — выберите один или несколько файлов.
- **Массовые действия:** **«Принять»** — файл появится у вас в списке; **«Отклонить»** — файл будет удалён из ожидающих.
- Также можно принять или отклонить каждый файл по отдельности кнопками в строке.

### 17.2 Вкладка «Общие папки»

- Отображает список общих папок, доступ к которым вам предоставили.
- Для каждой папки указано: название, пригласивший, уровень доступа.
- **Чекбоксы** — массовый выбор.
- **Массовые действия:** **«Принять»** — папка появится в разделе «Общие папки»; **«Отклонить»** — приглашение будет отклонено.

## 18. Администрирование

Раздел доступен только пользователям с ролью администратора. Позволяет управлять пользователями и отправлять уведомления.

### 18.1 Список пользователей

- Таблица со списком всех зарегистрированных пользователей.
- Столбцы: имя, email, статус, дата регистрации, **Уведомления** (Вкл / Выкл).

### 18.2 Уведомления

- **Уведомить пользователя:** выберите пользователя → нажмите **«Уведомить»** → укажите заголовок и текст сообщения. Если у пользователя отключены push-уведомления, сообщение будет отправлено на email.
- **Массовая рассылка:** нажмите **«Оповестить всех»** → укажите заголовок и текст. Сообщение будет отправлено всем пользователям (push или email — в зависимости от их настроек).
