# Логические ошибки и нестыковки — Фронтенд

## 1. Несоответствие ожидаемых полей между списком и детальной карточкой

**Проблема:** Фронтенд ожидает определённые поля от API, но бэкенд возвращает разные наборы полей в `buildFileCard()` (детальная карточка) и `buildListItem()` (список).

**Поля, которые есть в карточке, но отсутствуют в списке:**
- `is_owner` — используется в `file-detail.component.ts` для проверки прав
- `access_type` — используется в `shared-folders.component.ts`
- `is_favorite` — используется для отображения звезды
- `display_name` — используется для отображения
- `description` — используется в карточке
- `tags` — используются в карточке

**Стратегия:**
- Дополнить модель `FileListItem` недостающими полями
- Синхронизировать бэкенд (см. `04-backend-logical-errors.md` пункт 5)
- Проверить все места использования `FileListItem` на наличие обращений к отсутствующим полям

## 2. `formatSize()` — разный вывод на разных страницах

**Проблема:** Из-за разных реализаций `formatSize()` один и тот же файл может отображаться с разным размером на разных страницах.

**Примеры:**
- `file-detail.component.ts`: `1024 B`, `1.0 KB`, `1.0 MB`, `1.0 GB`
- `folders-tree.component.ts`: `1.0 КБ`, `1.0 МБ`, `1.0 ГБ`, `—` для нуля
- `file-list.component.ts`: `1.0 КБ`, `1.0 МБ` (нет поддержки B/ГБ ниже/выше порога)

**Стратегия:**
- См. `05-frontend-duplication.md` пункт 1 — единая функция во всех местах

---

## 3. `shared-folders.component.ts` — неверное определение `isVideo`

**Проблема:** В `shared-folders.component.ts` (строка ~225) проверка `file.type.startsWith('video/')` используется для решения, генерировать ли thumbnail. Но:
- `file.type` (File API) не всегда совпадает с `file.mime_type` (серверный)
- В `folders-tree.component.ts:937` — эта же проверка корректна, но в `shared-folders.component.ts` она может отличаться

**Стратегия:**
- Проверять `mime_type` с сервера, а не `file.type` с клиента, для согласованности
- Вынести определение необходимости thumbnail в `VideoThumbnailService`

## 4. `addModal` — upload и link в одном компоненте

**Проблема:** Модальное окно добавления (`addModal`) переключается между `file` и `link` табами (folders-tree:227), но:
- Состояние upload и link сбразываются только при закрытии модалки, не при переключении табов
- Если пользователь начал загрузку файла, затем переключился на link таб и обратно — прогресс загрузки потерян, но превью ссылки осталось

**Стратегия:**
- Сбрасывать состояние соответствующего таба при переключении
- Или сделать отдельные модальные окна для upload и link

## 5. `linkForm` — валидация URL

**Проблема:** В `folders-tree.component.ts:235` валидация URL: `Validators.pattern(/^https?:\/\/.+/)`. Но:
- Не пропускает `ftp://`, `ftps://` и другие протоколы
- Пропускает `http://` без домена (например, `http://`)

**Стратегия:**
- Использовать более точную валидацию URL
- Или довериться бэкенду (он всё равно перепроверит)

## 6. `matchMedia` отсутствует — `trackBy` не везде

**Проблема:** В шаблонах Angular 17+ рекомендуется использовать `@for` с `track` вместо `*ngFor` с `trackBy`. В проекте используются оба подхода.

**Места:**
- `admin.component.ts` — `*ngFor` с `trackById()`
- `support.component.ts` — `*ngFor` с `trackById()`
- Другие компоненты — `@for` с `track`

**Стратегия:**
- Привести к единому стилю: использовать `@for (item of items; track item.id)` везде

---

## 7. Отсутствие обработки ошибок в `loadFile()`

**Проблема:** В `file-detail.component.ts:180` — ошибка загрузки файла обрабатывается только `() => this.loading.set(false)`. Пользователь видит только вечную загрузку.

**Стратегия:**
- Добавить `error` сигнал и отображать сообщение об ошибке в UI
- Предусмотреть кнопку "Повторить"

---

## 8. `FolderNotes` — дублирование между shared и local folders

**Проблема:** `folderNotesOpen` (folders-tree:134) — единый сигнал для открытия заметок, но:
- В `folders-tree.component.html` используется один и тот же `ThreadCommentsComponent` и для local folder notes, и для shared folder discussion
- Контекст (`contextSharedFolderId`) не всегда передаётся корректно

**Стратегия:**
- Разделить на два отдельных сигнала/компонента: `localFolderNotes` и `sharedFolderDiscussion`

---

## 9. `editorExpanded` — дублирование через query param

**Проблема:** Редактор может быть открыт в expanded-режиме через query param `editor=expanded` (file-detail:134-138). Но `editorExpanded` — локальный сигнал, который не синхронизируется обратно в URL. При перезагрузке, если param уже не в URL — expanded теряется.

**Стратегия:**
- Синхронизировать сигнал с query param через Router
- Или не использовать query param для expanded, только локальный сигнал
